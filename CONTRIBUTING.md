# Contributing

Your contributions are always welcome!

## Guideline

* Add one link per Pull Request.
    * Make sure the PR title is in the format of `Add project-name`.
    * Write down the reason why the library is awesome.
* Add the link: `* [project-name](http://example.com/) - A short description ends with a period.`
    * Keep descriptions concise and **short**.
* Add a section if needed.
    * Add the section description.
    * Add the section title to Table of Contents.
* Search previous Pull Requests or Issues before making a new one, as yours may be a duplicate.
* Check your spelling and grammar.
* Remove trailing whitespace.

Both opensource as well as proprietary/commercial software can be listed. In case of commercial software it is 
important that the software actually already exists as a buyable product and not only at a future date.
